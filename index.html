Objective

In this lab, you will make something similar to the map above. We will use web mapping technologies, Mapbox, Leaflet, and GitHub. Mapbox is a provider of custom online maps for websites and applications such as Foursquare, Facebook, and The Weather Channel. Making a style on

  1. Go to Mapbox website - https://www.mapbox.com/
  2. Sign up for a new account. If you can't find a link here you go - Sign Up Here!
  3. Choose the 'Pay as you Go' plan. This will give you the free tier and plenty of loads to learn how to use MapBox.
  4. Once you are signed up and logged in, go to the MapBox Studio to start design your own custom basemap. https://studio.mapbox.com/
  5. In the studio prompt click on 'new style'
  7. Choose a template, I choose 'Dark' for this lab, but spice it up select whichever one you want.
  8. Once you select the base template, you want to start with click on 'customize template' to get into MapBox Studio
  9. This part you need to experiment with and have fun. As you can see from the Table of Contents, that each layer is there and you can click on them to open a panel customize them. You can change fonts, colors, line weights, etc. If you click on a title of the layer, you can drag them to rearrange drawing order. You can also select a layer and delete them. As for me, I am going to add a layer to the mix, the terrain layer, and mix it in with the default dark template to make 'Dark Terrian'.
  10. To add a new layer click the 'add layer' button
  11. A panel will popup. I am going to choose to add Terrian (RBG encoded DEM)
  12. This will add the layer to the top of the table of contents.
  13. Drag the layer down in the drawing order so the roads and labels will draw over it. Be sure to click out of the panel to see the result.
  14. Once you are done modifying your Mapbox Style, it is time to publish it. Click the Publish button to put it on the production server and have it ready for your use.
  15. You should get a prompt saying you have succesfully published your style.
  16. Now click the share button. This will give you two essential pieces of information that you will use afterwards during coding, Your Style URL and access token. For now, keep access to this information on hand.

Make your Map live on

  1. GitHub is not only a great place to store and share your code with others but they also offer free web hosting of your HTML, CSS, and JavaScript projects!
  2. First thing‚Äôs first: get yourself a GitHub account!https://github.com/join
  3. Download and install the GitHub Desktop app. https://desktop.github.com/ This is the app that we‚Äôll use to get our code up on GitHub. Open the downloaded application file and continue through the installation process as required with your computer‚Äôs operating system.
  4. Before we upload our code to GitHub we first need to make GitHub Desktop aware of our project files.
  5. Open up the GitHub Desktop app and Sign in.
  6. Then Click the ‚ÄúCreate New Repository‚Äù button.
  7. When the ‚ÄúCreate a New Repository‚Äù dialog window appears, fill in a name for your map. I used DarkTerrian.
  8. The ‚ÄúLocal Path‚Äù text input is where your repository will live on your computer; this is not where your website files currently reside on your computer‚Äôs hard drive. GitHub Desktop needs to create a new, empty folder.
  9. We‚Äôll need to copy your site files over later, but for now, select a folder such as /Sites in your home folder. This will create a new folder inside the /Sites folder with the name of your repository.
  10. Leave all the other options in the form at their default settings and click the ‚ÄúCreate Repository‚Äù button.
  11. Copy all your files from the site your made to the new Repository folder
  12. The Github Desktop app will now see the changes to the folder
  13. With all our files in the GitHub Repository folder, we‚Äôre ready to commit our files! To ‚Äúcommit‚Äù files means to take a ‚Äúsnapshot‚Äù of the files and folders in their current state. By doing so, we get them ready to be pushed up to a repository on GitHub. Before we commit our files for publishing, we first need to tell GitHub what this commit is all about. Fill in the ‚ÄúSummary‚Äù text input, located near the bottom left of the user interface, with a high-level title of what this commit entails; something simple like, ‚ÄúMy first commit! üòÄ‚Äù works just fine in this instance. Now click the ‚ÄúCommit to master‚Äù button!
  14. fter clicking the ‚ÄúCommit to master‚Äù button in the previous step, you may be wondering: What just happened? My files are gone and the app is empty now! Don‚Äôt fret! In the bottom left corner of the GitHub Desktop interface there‚Äôs a little indicator that your commit was successful.
  15. Now that our files are committed, this allows us to push our commit to GitHub! To ‚Äúpush‚Äù files means to upload the committed files in their current state to a repository on GitHub. Find and click the ‚ÄúPublish repository‚Äù button along the top of the GitHub Desktop interface.
  16. You will now find the Pusblish window. All the options in the form are fine as-is except the ‚ÄúKeep this code private‚Äù checkbox; this needs to be unchecked in order for you to share your site with friends and family.
  17. Click the ‚ÄúPublish Repository‚Äù button to send your files up to GitHub!
  18. Now, in your browser, go to GitHub.com and find your repository page to make sure your files made the journey to their new home.
  19. On your repository page, click on the settings and scroll to your GitHub Pages setting. Choose the setting for Source and select Master Branch.
  20. You will now see your URL for you webmap
  21. Open the URL in your browser and see your map live online.

You are done!
